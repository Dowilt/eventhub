# EventHub — Планировщик мероприятий

Веб-приложение для планирования и управления мероприятиями. Реализовано в рамках лабораторной работы №5.

## Функциональность

- **Просмотр событий** — карточки мероприятий в виде сетки (2 в ряд)
- **Фильтрация** — по категориям (Учёба, Спорт, Развлечения, Работа, Личное) через Chip-фильтры
- **Добавление события** — через FAB-кнопку и форму (название, описание, место, категория, дата, время, участники)
- **Детали события** — полная информация с раскрываемыми блоками (Описание, Участники)
- **Удаление события** — кнопка удаления на карточке с возможностью отмены через SnackBar
- **Редактирование события** — кнопка на экране деталей, предзаполненная форма
- **Поиск** — поиск по названию в реальном времени
- **Сортировка** — по дате, названию или категории
- **Статистика** — отдельный экран с круговыми индикаторами по категориям, ближайшие 3 события

## Запуск

```bash
docker-compose up -d --build
```

Приложение будет доступно по адресу: **http://localhost**

API доступно по адресу: **http://localhost/api/events**

## Остановка

```bash
docker-compose down
```

Данные сохраняются в Docker volume `pgdata` и не теряются при остановке/запуске.

## API эндпоинты

| Метод  | URL                    | Описание               |
|--------|------------------------|------------------------|
| GET    | /api/events            | Список всех событий    |
| GET    | /api/events/{id}       | Одно событие           |
| POST   | /api/events            | Создать событие        |
| PUT    | /api/events/{id}       | Обновить событие       |
| DELETE | /api/events/{id}       | Удалить событие        |
| GET    | /api/categories        | Список категорий       |
| GET    | /api/stats             | Статистика             |

## Демонстрация

Для демонстрации:
1. Запустите `docker-compose up -d --build`
2. Откройте http://localhost в браузере
3. Протестируйте все функции: фильтрация, поиск, добавление, редактирование, удаление, статистика
